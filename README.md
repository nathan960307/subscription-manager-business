# Business Server

구독 관리 및 자동 생성 서비스

---

## 프로젝트 개요

사용자의 결제 데이터를 기반으로 구독 서비스를 생성하고 관리하는 서버로,  
반복 결제 패턴을 분석하여 자동으로 구독을 생성하는 비즈니스 로직을 담당합니다.

본 서버는 인증 서버와 분리된 구조에서 핵심 비즈니스 로직을 처리합니다.

---

## 기술 스택
- Java 21
- Spring Boot
- MyBatis
- MySQL
- Redis
- Docker

---

## 주요 기능

- 구독 생성, 조회, 수정, 삭제 (CRUD)
- 결제 이력 관리 및 조회
- 반복 결제 분석을 통한 구독 자동 생성
- 서비스명 정규화 처리
- 구독 변경 이력 관리
- 월 단위 구독 비용 및 결제 예정일 통계 제공

---

## 아키텍처

    [ Auth Server ]
         ↓
    [ Business Server ] → MySQL

---

## 서비스 흐름

1. 외부 결제 데이터 조회 (Mock Data 기반)
2. 결제 내역을 서비스명 기준으로 그룹화
3. 반복 결제 패턴 분석
4. 구독 후보 필터링
5. 서비스명 정규화 처리
6. 구독 자동 생성 및 저장
7. 결제 이력과 구독 연결

---

## 설계 포인트

- 반복 결제 기반 구독 자동 생성 로직 설계
- 서비스명 정규화 테이블을 통한 데이터 일관성 확보
- 구독 / 결제 / 이력 테이블 분리 설계
- 확장 가능한 구조 (외부 API 연동 고려)

---

## 트러블슈팅

### 반복 결제 판단 로직 문제

- 문제  
  반복 결제 기준이 명확하지 않아 구독이 중복 생성되거나 누락되는 문제 발생

- 해결  
  결제 간격, 반복 횟수, 결제 주기를 기준으로 판별 로직을 재설계하고  
  서비스명 정규화 테이블을 도입하여 정확도를 개선

- 결과  
  구독 자동 생성 정확도 향상 및 데이터 일관성 확보
